【vue组件开发日志】第5天：页码器组件 Paginator.vue 需求分析 2017-07-07

发表于 2017-07-08
明确需求，稳扎稳打

sublime 上编辑 evernote 笔记折腾了 1 个小时…

一、功能

  1. 首页（可选）
  2. 上一页（必选）
  3. 翻页（必选）
    - 点击翻页（可选，直接点击页码）
    - 手动翻页（可选，点击 [ … ] 以找到所需页码）
    - 输入翻页（可选）
    - 下拉翻页（可选）
  4. 下一页（必选）
  5. 末页（可选）

二、样式

  1. 电脑端（所有功能）

    - 显示页码数 >= 总页数 （此处假设 50 >= 5）
      [ << ] [ < ] [ 1 ] [ 2 ] [ 3 ] [ 4 ] [ 5 ] [ > ] [ >> ]

    - 显示页码数 < 总页数 （此处假设 5 < 50）
      - 手动翻页
        [ << ] [ < ] [ 1 ] [ 2 ] [ 3 ] [ 4 ] [ 5 ] [ … ] [ > ] [ >> ]

      - 输入翻页
        [ << ] [ < ] [ enter_page_no ] [ > ] [ >> ]

      - 下拉翻页
        [ << ] [ < ] [ dropdown_pageno[v] ] [ > ] [ >> ]

      - 输入翻页 + 手动翻页 （上下位置可互换）
        [ << ] [ < ] [ 1 ] [ 2 ] [ 3 ] [ 4 ] [ 5 ] [ … ] [ enter_page_no ] [ > ] [ >> ]

      - 下拉翻页 + 手动翻页 （上下位置可互换）
        [ << ] [ < ] [ 1 ] [ 2 ] [ 3 ] [ 4 ] [ 5 ] [ … ] [ dropdown_pageno[v] ]> [ > ] [ >> ]

  1. 移动端（所有功能）

    和电脑端（所有功能）一致，只是页码器宽度要自适应屏幕宽度.

三、定制

  1. 功能定制

    - 是否启用可选功能。可选功能已经在第一节（功能）中说明。

  2. 样式定制

    如果有选择主题，样式定制将覆盖相应的主题样式。

    样式可定制的基本准则：

      1. 哪些样式可定制？——公共部件的样式；

      2. 什么是公共部件？——对整体组件的抽象提取后得到的部件或部件列表；

      3. 为什么？——确保整体组件的外观是和谐的。

    根据准则，提取公共部件如下：（打 * 号为默认值）

      1. 框型部件：[ << | < | page_no | > | >> ] 可定制如下样式：

        - 形状 shape: ‘square’* | ‘circle’ // 方形自带圆角

        - 大小 size: initial* |’any_px_value’ // px 通过 px2em 转为 em

        - 填充颜色 fillColor: initial* | ‘any_color_code’

        - 文本颜色 textColor: initial* | ‘any_color_code’

        - 文本大小 textSize: initial* | ‘any_px_value’

      2. 输入部件：[ enter_page_no ] 可定制如下样式：

        - 继承框型部件样式 inhert: true* | false // 除了形状 shape 不能继承

        - 占位符 placeholder: ‘current_page_no/total_page_no’* | ‘any_placeholder_text’ // 中间的斜杠是斜杠，不是或

        - 位置 position: ‘lower’* | ‘upper’ // 见 二、样式 输入翻页 + 手动翻页

      3. 下拉部件：[ dropdown_pageno[v] ] 可定制如下样式：

        - 继承框型部件样式 inhert: true* | false // 除了形状 shape 不能继承

        - 占位符 placeholder: ‘current_page_no/total_page_no’* | ‘any_placeholder_text’ // 中间的斜杠是斜杠，不是或

        - 位置 position: ‘lower’* | ‘upper’ // 见 二、样式 下拉翻页 + 手动翻页

  3. 主题定制

  主题就是一系列默认样式定制。选择主题后，可通过上述样式定制来修改主题。

四、接口

  1. 参数：父组件 -> 子组件

    - 总页数 totalPageNo: 0*
    - 分页尺寸 pageSize: 8*
    - 初始页码 initPageNo: 1*

  2. 事件：子组件 -> 父组件

    - 翻页事件 onPageNoChange 带参数 currentPageNo 为当前页码。
